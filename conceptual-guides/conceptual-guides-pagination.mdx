The BookHub API limits results per request, so retrieving a full dataset requires multiple requests.

When you request a list (e.g., `GET /v2/books`), the API returns the total number of items, the total number of pages, and the current page. See [Handle pagination](/how-to-guides/how-to-guides-handle-pagination) for implementation details.

The server automatically calculates `totalPages` from `totalItems` and `itemsPerPage`; you do not need to compute it. For example, if there are 250 books and the page size is 100, `totalPages` will be 3. The first two pages will return 100 items each, and the last page will return the remaining 50 items. If `page` exceeds `totalPages`, the API returns `200 OK` with an empty `books` array.


```json
{
  "status": "success",
  "data": {
    "books": [
      {
        "bookId": "abc123",
        "title": "The Great Adventure",
        "author": "Jane Smith"
      }
    ],
    "pagination": {
      "totalItems": 250,
      "totalPages": 3,
      "currentPage": 1,
      "itemsPerPage": 100
    }
  }
}
```

The `GET /v2/books` endpoint also accepts `status`, `sort`, and `order` parameters. See [API endpoints](/reference/reference-api-endpoints#get-all-books) for details.

## Pagination parameters

Pass `page` and `limit` as query parameters to control which page you receive:

```http
GET /v2/books?page=2&limit=100
```

The `page` parameter defaults to `1`, and `limit` defaults to `20` with a maximum of `100`. See [API endpoints](/reference/reference-api-endpoints#get-all-books) for the full parameter reference.
