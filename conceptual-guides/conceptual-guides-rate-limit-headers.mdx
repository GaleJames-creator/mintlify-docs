By checking these headers in your API responses, you can track how many requests remain and when the limit will reset. This allows you to implement logic to slow down or pause requests if you're approaching the limit.

The BookHub API includes the following headers in each response to help you monitor your rate limit status:

* `X-RateLimit-Limit`: The total number of requests allowed in the current time window (e.g., 150).
* `X-RateLimit-Remaining`: The number of requests remaining in the current time window.
* `X-RateLimit-Reset`: The time when the current rate limit window resets, in UTC epoch seconds.

By tracking these headers, you can ensure that your application stays within the allowed rate limits and avoids being throttled by the API. This allows you to implement logic to slow down or pause requests if you're approaching the limit.
