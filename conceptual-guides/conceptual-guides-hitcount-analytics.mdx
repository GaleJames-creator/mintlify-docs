This metric helps publishers:

* **Gauge interest** in specific titles
* **Identify trending books** before they become bestsellers
* **Measure marketing effectiveness** by post-campaign views
* **Make data-driven decisions** about inventory and pricing

## How it works

The `hitCount` field is read-only and has an initial value of `0` when a book is created. Each time a customer retrieves the book's details via the `GET /v2/books/{bookId}` endpoint, the `hitCount` increments by `1`. This count is cumulative and persists throughout the book's lifetime, allowing you to track its performance over time.

The `hitCount` field is updated in real-time and reflects the total number of times a book's details have been accessed via the API. Update latency is minimal, but there may be slight delays during periods of high traffic. Client-side caching using `If-None-Match` and ETag headers returns a `304 Not Modified` response without incrementing `hitCount`, since no new API call reaches the server.

Repeated requests from the same customer will increment the `hitCount` each time, as it counts API calls rather than unique users. However, requests from publishers and cached responses do not affect the `hitCount`. 

Endpoint coverage for `hitCount`:

| Endpoint | `hitCount` field in response? |
| --- | --- |
| `GET /v2/books/{bookId}` | Yes |
| `GET /v2/books` | No |
| `POST /v2/books` | No |
| `DELETE /v2/books/{bookId}` | No |
| `PATCH /v2/books/{bookId}/finalize` | No |

For error responses and rate limit details, see [API endpoints](/reference/reference-api-endpoints#error-codes).

When you retrieve a book's details using the `GET /v2/books/{bookId}` endpoint, the response includes the `hitCount` field, allowing you to see how many times that book's details have been viewed via the API. For example:

```json
{
  "status": "success",
  "data": {
    "bookId": "abc123",
    "title": "The Great Adventure",
    "author": "Jane Smith",
    "hitCount": 1247
  }
}
```

## Use cases for `hitCount`

<Note>The following Python examples are for illustrative purposes. Actual implementation may vary based on your programming language and setup.</Note>

* **Trending analysis**: Identify which books are getting the most attention. The following example retrieves all books and sorts them by `hitCount` to find the top 10 most-viewed titles:
  
   ```python
   all_books = fetch_all_books()
   trending = sorted(all_books, key=lambda x: x['hitCount'], reverse=True)[:10]
   ```

* **Marketing ROI**: Compare hit counts before and after marketing campaigns. The following example measures the increase in views for a book after running a marketing campaign:
  
   ```python
   before_campaign = get_book('abc123')['hitCount']
   after_campaign = get_book('abc123')['hitCount']
   increase = after_campaign - before_campaign
   print(f"Campaign generated {increase} additional views")
   ```

* **Inventory planning**: Prioritize books with high view counts for restocking. The following example identifies books with a `hitCount` above a certain threshold and notifies the warehouse to restock:
  
   ```python
   if book['hitCount'] > 1000 and book['status'] == 'ACTIVE':
       notify_warehouse_to_restock(book['bookId'])
   ```
