The `hitCount` field in the `BookWithAnalytics` schema provides a count of how many times a book's details have been retrieved via the `GET /v2/books/{bookId}` endpoint. This can be used to track the popularity and performance of books over time. 

Only buyer requests increment `hitCount` — publisher requests and cached responses do not. For details on `hitCount` data type, initial value, update behavior, and availability, see [API endpoints](/reference/reference-api-endpoints#hitcount-field-explained).

## Prerequisites

<Prerequisites />

* Familiarity with the BookHub Publisher API v2.
* A valid JWT for authentication.
* Access to your codebase that integrates with the BookHub Publisher API.

## Step 1: Retrieve book details with `hitCount`

To see the `hitCount` for a specific book, make a GET request to the `GET /v2/books/{bookId}` endpoint. 

```python
import requests

BASE_URL = "https://api.bookhub.com/api"
TOKEN = "YOUR_JWT_TOKEN"
HEADERS = {"Authorization": f"Bearer {TOKEN}"}

response = requests.get(
    f"{BASE_URL}/v2/books/{book_id}",
    headers=HEADERS
)
response.raise_for_status()
book = response.json()['data']
hit_count = book.get('hitCount', 0)
print(f"{book['title']} ({book['bookId']}): {book['hitCount']} views")

```

The response will include the `hitCount` field, which indicates how many times that book's details have been accessed via the API. For example:

```json

  "data": {
```

## Step 2: Retrieve `hitCount` for all books and identify top performers

The `GET /v2/books` endpoint does not include `hitCount` in its response. To retrieve `hitCount` for all books, fetch each book individually using `GET /v2/books/{bookId}`.

<Important>This approach makes one API request per book. For large catalogs, this may hit rate limits (150 requests per minute). See [Monitor rate limits](/how-to-guides/how-to-guides-monitor-rate-limits) for guidance.</Important>

```python
# Fetch all books to get bookId values
list_response = requests.get(
    f"{BASE_URL}/v2/books",
    headers=HEADERS
)
list_response.raise_for_status()
books = list_response.json()['data']['books']

# Fetch hitCount for each book individually
book_performance = []
for book in books:
    detail_response = requests.get(
        f"{BASE_URL}/v2/books/{book['bookId']}",
        headers=HEADERS
    )
    detail_response.raise_for_status()
    detail = detail_response.json()['data']
    book_performance.append({
        'title': detail['title'],
        'bookId': detail['bookId'],
        'hitCount': detail.get('hitCount', 0)
    })

# Sort by hitCount descending
book_performance.sort(key=lambda x: x['hitCount'], reverse=True)

for book in book_performance:
    print(f"{book['title']} ({book['bookId']}): {book['hitCount']} views")
```

The output lists books sorted by performance:

```text
The Great Adventure (abc123): 1247 views
Mystery at Midnight (def456): 980 views
Cooking with Love (ghi789): 450 views
```

## Step 3: Set a performance threshold

To identify high-performing books, you can set a threshold for the `hitCount`. For example, you might want to flag any book with a `hitCount` greater than 1000 as a high performer. You can implement this logic in your code as follows:

```python
for book in book_performance:
    status = "⭐ High performer" if book['hitCount'] > 1000 else ""
    print(f"{book['title']} ({book['bookId']}): {book['hitCount']} views {status}")
```
