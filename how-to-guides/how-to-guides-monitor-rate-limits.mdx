Monitor rate limit headers in API responses to avoid exceeding the BookHub Publisher API limit of 150 requests per minute. Exceeding the limit returns a `429 Too Many Requests` response.

## Prerequisites

<Prerequisites />

* Familiarity with BookHub Publisher API v2.
* A valid JWT for authentication.
* Access to your codebase that integrates with the BookHub Publisher API.

## Step 1: Track rate limit headers

Every API response includes three rate limit headers: `X-RateLimit-Limit`, `X-RateLimit-Remaining`, and `X-RateLimit-Reset`. For details on each header, see [API endpoints](/reference/reference-api-endpoints#get-all-books-response-headers).

Add a rate limit check to your API calls:

```python
import requests
import time

BASE_URL = "https://api.bookhub.com/api"
TOKEN = "YOUR_JWT_TOKEN"
HEADERS = {"Authorization": f"Bearer {TOKEN}"}

def check_rate_limits(response):
    limit = int(response.headers.get('X-RateLimit-Limit', 150))
    remaining = int(response.headers.get('X-RateLimit-Remaining', limit))
    reset = int(response.headers.get('X-RateLimit-Reset', 0))
    
    print(f"Rate limit: {remaining}/{limit} remaining")
    
    if remaining < 10:
        # Calculate seconds until reset from Unix timestamp
        wait_seconds = min(60, max(1, reset - int(time.time())))
        print(f"Low on rate limit — resets in {wait_seconds} seconds")
    
    return remaining
```

## Step 2: Handle 429 responses

When you exceed the rate limit, the API returns `429 Too Many Requests`. Implement exponential backoff to retry automatically:

```python
def make_request_with_backoff(url, max_retries=3):
    for attempt in range(max_retries):
        response = requests.get(url, headers=HEADERS)
        
        if response.status_code == 429:
            reset = int(response.headers.get('X-RateLimit-Reset', 0))
            wait_seconds = min(60, max(1, reset - int(time.time())))
            print(f"Rate limit exceeded. Waiting {wait_seconds} seconds...")
            time.sleep(wait_seconds)
            continue
        
        response.raise_for_status()
        check_rate_limits(response)
        return response
    
    raise Exception(f"Failed after {max_retries} retries")
```

## Step 3: Test your implementation

Make a request and verify the rate limit headers are present in the response:

```bash
curl -X GET "https://api.bookhub.com/api/v2/books" \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -v
```

Confirm the response includes `X-RateLimit-Limit`, `X-RateLimit-Remaining`, and `X-RateLimit-Reset` headers. The `X-RateLimit-Reset` value is a Unix timestamp — convert it to a human-readable time if needed:

```python
import datetime
reset_time = datetime.datetime.fromtimestamp(reset)
print(f"Rate limit resets at: {reset_time}")
```

## Response

A `429 Too Many Requests` response indicates the rate limit has been exceeded:

```json
{
  "status": "error",
  "message": "Rate limit exceeded"
}
```

The response also includes the rate limit headers so you can calculate when to retry.
