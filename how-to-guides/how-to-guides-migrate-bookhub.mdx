This guide walks you through migrating your code from BookHub Publisher API v1 to v2. It covers the two breaking changes in v2: the `hitCount` field addition and the removal of title sorting. For the complete list of changes, see [Breaking changes](/reference/reference-breaking-changes). For a complete overview, see [Release notes](./reference/reference-release-notes).

## Prerequisites

<Prerequisites />

* Familiarity with BookHub Publisher API v1.
* Access to your codebase that integrates with the BookHub Publisher API.
* A valid JWT for authentication.

## Step 1: Update base URLs

Start by updating all API URLs in your code from v1 to v2. For example:

### Before (v1)

```http
https://api.bookhub.com/api/v1/books
```

### After (v2)

```http
https://api.bookhub.com/api/v2/books
```

## Step 2: Handle the `hitCount` field

Use the `hitCount` field on individual books to track views. Replace separate analytics API calls with this field in your existing code for retrieving book details. For details on `hitCount` data type, range, and availability, see [API endpoints](/reference/reference-api-endpoints#get-single-book).

### Before (v1)

```python
response = requests.get(f"{BASE_URL}/v1/books/{book_id}")
book = response.json()['data']
# book has no hitCount field
```

### After (v2)

```python
response = requests.get(f"{BASE_URL}/v2/books/{book_id}")
book = response.json()['data']
hit_count = book.get('hitCount', 0)  # New field available
print(f"This book has been viewed {hit_count} times")
```

## Step 3: Replace title sorting

If your current implementation sorts books by title, adjust your approach using one of these options in BookHub Publisher API v2:

### Option A: Use date sorting (recommended)

### Before (v1)

```python
# Sort by title in ascending order
response = requests.get(f"{BASE_URL}/v1/books?sort=title&order=asc")
```
### After (v2)

```python
# Sort by creation date in descending order
response = requests.get(f"{BASE_URL}/v2/books?sort=createdDate&order=desc")
```

### Option B: Implement client-side sorting

```python
# Fetch all books
response = requests.get(f"{BASE_URL}/v2/books?limit=100")
books = response.json()['data']['books']

# Sort by title on client side
sorted_books = sorted(books, key=lambda x: x['title'].lower())
```

## Step 4: Update error handling

Be ready to handle the new error triggered when attempting to sort by title.

A `400` error occurs specifically when `sort=title` is passed because v2 no longer supports title sorting. The error response looks like this:

```json
{
  "status": "error",
  "message": "Invalid sort parameter",
  "errors": [{
    "field": "sort",
    "error": "INVALID_VALUE",
    "description": "v2 only supports sorting by createdDate. Use v1 for title sorting."
  }]
}
```

Then handle it in your code:

```python
BASE_URL = "https://api.bookhub.com/api"

def get_books(sort_by='createdDate', order='desc', retry=True):
    # retry=False prevents infinite loops if the fallback also fails 
    params = {'sort': sort_by, 'order': order}
    response = requests.get(f"{BASE_URL}/v2/books", params=params)
    
    if response.status_code == 400 and retry:
        error_data = response.json()
        errors = error_data.get('errors', [])
        for error in errors:
            if error.get('field') == 'sort':
                print(f"Invalid sort parameter: {error.get('description')}")
                return get_books(sort_by='createdDate', order=order, retry=False)
    
    return response.json()
```

## Step 5: Test your migration

After making these changes, test your code to ensure it works correctly with BookHub Publisher API v2. Verify that the `hitCount` field is accessible and that sorting behaves as expected.

1. Send a Create book request and verify:

    * The responsereturns `201 Created` with a `bookId`.
    * The Finalize book request returns `200 OK` and updates the book status to `ACTIVE`.
    * The `hitCount` field is included and updates correctly for each retrieved book.
    * The date sorting functions as expected.

For example:

```python
import requests

BASE_URL = "https://api.bookhub.com/api"
TOKEN = "YOUR_JWT_TOKEN"
HEADERS = {"Authorization": f"Bearer {TOKEN}"}

# Step 1. Create a book
create_response = requests.post(
    f"{BASE_URL}/v2/books",
    headers={**HEADERS, "Content-Type": "application/json"},
    json={
        "title": "My New Book",
        "author": "John Doe",
        "description": "A new biography from a well-known author",
        "language": "en-US",
        "publisher": "My Publishing House",
        "publishedDate": "2026-03-15",
        "isbn": "978-3-16-148410-0",
        "price": 24.99,
        "coverImages": ["https://example.com/cover.jpg"],
        "genres": ["Fiction", "Adventure"],
        "bookFormat": "Hardcover"
    }
)
print(f"Create: {create_response.status_code}")
book_id = create_response.json()['data']['bookId']

# Step 2. Finalize the book
finalize_response = requests.patch(
    f"{BASE_URL}/v2/books/{book_id}/finalize",
    headers={**HEADERS, "Content-Type": "application/json"},
    json={"notes": "Migration test"}
)
print(f"Finalize: {finalize_response.status_code}")

# Step 3. Retrieve the book and verify hitCount
get_response = requests.get(
    f"{BASE_URL}/v2/books/{book_id}",
    headers=HEADERS
)
book = get_response.json()['data']
print(f"Status: {book['status']}")
print(f"hitCount: {book.get('hitCount', 'field not found')}")

# Step 4. Verify date sorting works
list_response = requests.get(
    f"{BASE_URL}/v2/books",
    headers=HEADERS,
    params={"sort": "createdDate", "order": "desc"}
)
print(f"List books: {list_response.status_code}")
```
