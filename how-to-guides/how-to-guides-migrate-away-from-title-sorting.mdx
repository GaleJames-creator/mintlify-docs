The `GET /v2/books` endpoint no longer supports the `sort=title` parameter. Replace it with `sort=createdDate` or implement client-side sorting.

## Prerequisites

<Prerequisites />

* Familiarity with BookHub Publisher API v1.
* Access to your codebase that integrates with the BookHub Publisher API.
* A valid JWT for authentication.

## Step 1: Audit your codebase for `sort=title` parameters

```bash
# Search your codebase
grep -r "sort=title" .
grep -r "sort.*title" .
```

## Step 2: Replace with `sort=createdDate` or client-side sorting

To update your code, replace any instances of `sort=title` with either `sort=createdDate` or client-side sorting. Option A uses the API's `sort=createdDate` parameter to return results pre-sorted. Option B fetches unsorted results and sorts them in your application code. Here are examples of both approaches:

### Before (v1)

```python
response = requests.get(f"{BASE_URL}/v1/books", params={"sort": "title", "order": "asc"})
```

### After (v2) option A: `sort=createdDate` (recommended)

```python
response = requests.get(f"{BASE_URL}/v2/books", params={"sort": "createdDate", "order": "desc"})
```

### After (v2) option B: client-side sorting

<Important>Client-side sorting requires fetching all books. For large catalogs, this may hit rate limits (150 requests per minute) or impact performance. Use `sort=createdDate` where possible.</Important>

When fetching books, you can sort them by title on the client side after retrieving the data. Here's an example of how to do this in Python:

<Note>This example fetches up to 100 books in a single request. For catalogs larger than 100 books, paginate through all pages before sorting. See [Handle pagination](/how-to-guides/how-to-guides-handle-pagination) for details.</Note>

```python
import requests

def get_books_sorted_by_title(ascending=True):
    # Fetch all books (or paginate through all pages)
    response = requests.get(
        "https://api.bookhub.com/api/v2/books",
        params={"limit": 100},
        headers={"Authorization": f"Bearer {token}"}
    )
    response.raise_for_status()
    books = response.json()['data']['books']
    
    # Sort by title on client side
    return sorted(
        books,
        key=lambda x: x.get('title', '').lower(),
        reverse=not ascending
    )
```

## Response

A successful `GET /v2/books` request using `sort=createdDate` returns `200 OK`:

```json
{
  "status": "success",
  "data": {
    "books": [
      {
        "bookId": "abc123",
        "title": "The Great Adventure",

        "bookId": "def456",
        "title": "Mystery at Midnight",
        "author": "Bob Johnson",
        "status": "ACTIVE",
        "price": 19.99,
        "createdDate": "2024-12-24T14:20:00Z",
```

Passing `order=desc` returns books newest first, `order=asc` for oldest first. If your code still passes `sort=title`, v2 returns a `400` error:

```json
{
  "status": "error",
  "message": "Invalid sort parameter",
  "errors": [{"field": "sort", "error": "INVALID_VALUE"}]
}
```

See [API endpoints](/reference/reference-api-endpoints#get-all-books) for all available parameter values and the full error response.

## Step 3: Test your changes

Send a test request and confirm the response returns `200 OK` with books ordered by `createdDate`. For catalogs larger than 100 books, run the pagination test from [Handle pagination](/how-to-guides/how-to-guides-handle-pagination) to verify all pages return correctly.

```bash
curl -X GET "https://api.bookhub.com/api/v2/books?sort=createdDate&order=desc" \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"
```

Verify the first book in the response has the most recent `createdDate`.
