---
title: 'Cache book details for faster performance'
description: 'Learn how to use ETag and Cache-Control headers to cache book details and reduce API calls.'
---
The `GET /v2/books/{bookId}` endpoint returns `ETag` and `Cache-Control` headers that can be used to cache book details and reduce unnecessary API calls. By implementing caching, you can improve performance and reduce latency when retrieving book information.

Here's how to use them:

```python
import requests

# First request
response = requests.get(f"{BASE_URL}/books/{book_id}")
etag = response.headers.get('ETag')
book_data = response.json()

# Later request with conditional GET
response = requests.get(
    f"{BASE_URL}/books/{book_id}",
    headers={'If-None-Match': etag}
)

if response.status_code == 304:
    # Use cached data
    print("Data unchanged, using cache")
else:
    # Update cache
    book_data = response.json()
```
