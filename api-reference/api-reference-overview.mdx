The BookHub Publisher API enables programmatic management of book inventory on the BookHub marketplace. This reference covers the available endpoints, request and response formats, and error handling.

## Versioning

The API uses URI path-based versioning (see also [Versioning](/reference/reference-breaking-changes#versioning)). The following versions are currently available:

- **v1**: Initial stable release — `https://api.bookhub.com/api/v1`
- **v2**: Current version with analytics features — `https://api.bookhub.com/api/v2`

## Authentication

All API requests require authentication using a JWT bearer token. For details, see the [Authentication](/reference/reference-authentication) section.

## Endpoints

The following endpoints are available in v2:

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/books` | `POST` | Create a new book |
| `/books` | `GET` | Get all books |
| `/books/{bookId}` | `GET` | Get a specific book |
| `/books/{bookId}` | `DELETE` | Delete a book |
| `/books/{bookId}/finalize` | `PATCH` | Finalize a book |

See [Endpoints](/api-reference/endpoint/createBook) for interactive endpoint documentation and request examples.

## Error handling

The API uses standard HTTP status codes to indicate success or failure. Error responses return a JSON object with a `status` and `message` field. For a complete list of error codes, see [API endpoints](/reference/reference-api-endpoints).

## Rate limiting

All versions support 150 requests per minute. Exceeded requests return a `429` response with rate limit headers. For more information, see the [API endpoints](/reference/reference-api-endpoints) and [monitor rate limits](/reference/reference-rate-limits) sections.
