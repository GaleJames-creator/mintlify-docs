The BookHub Publisher API enables programmatic management of book inventory on the BookHub marketplace. This reference covers the available endpoints, request and response formats, and error handling.

## Versioning

The API uses URI path-based versioning. The following versions are currently available:

- `v1`: Initial stable release &mdash; `https://api.bookhub.com/api/v1`
- `v2`: Current version with analytics features &mdash; `https://api.bookhub.com/api/v2`

## Authentication

All API requests require authentication using a JWT. For details, see [Authentication](/reference/reference-authentication).

## Endpoints

The following endpoints are available in v2:

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/books` | `POST` | Create a new book |
| `/books` | `GET` | Get all books |
| `/books/{bookId}` | `GET` | Get a specific book |
| `/books/{bookId}` | `DELETE` | Delete a book |
| `/books/{bookId}/finalize` | `PATCH` | Finalize a book |

See the [API Reference](/api-reference/endpoint/createBook) tab for the interactive endpoint playground with live request examples.

## Error handling

The API returns the following status codes: `200 OK`, `201 Created`, `204 No Content`, `400 Bad Request`, `401 Unauthorized`, `404 Not Found`, and `429 Too Many Requests`. Error responses include a `status` and `message` field, and an `errors` array for field-level validation failures. For details, see [API endpoints](/reference/reference-api-endpoints#error-codes).

## Rate limiting

All versions support 150 requests per minute. Exceeded requests return a `429` response with rate limit headers. For more information, see the [API endpoints](/reference/reference-api-endpoints) and [Rate limit headers](/reference/reference-rate-limit-headers) sections.
