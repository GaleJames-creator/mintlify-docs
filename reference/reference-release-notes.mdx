## Version 2.0 &mdash; January 2, 2026

### Hit count analytics

The `hitCount` field indicates how many times customers retrieve a book’s details via the `GET /v2/books/{bookId}` endpoint. Use `hitCount` to measure interest in specific titles, such as tracking post-campaign views to assess marketing impact, and make data-driven decisions about inventory and pricing.

See [Using the `hitCount` field for analytics](/conceptual-guides/conceptual-guides-hitcount-analytics), and [Use `hitCount` to track book performance](/how-to-guides/how-to-guides-use-hitcount-to-track-book-performance) for more details on using this new field effectively.

### Delete a book

Publishers can now permanently remove `INACTIVE` books from the BookHub marketplace using the `DELETE /v2/books/{bookId}` endpoint. Deleting a book removes it from search results and future queries, reducing catalog clutter. A successful delete returns `204 No Content` with no response body. For details on using this endpoint, see [Delete a book](/reference/reference-api-endpoints#delete-a-book).

### Known issues

The BookHub Publisher API v2 has the following known issues that will be addressed in future versions:

**`validation_context` and `provided_value` use snake_case**

The `validation_context` and `provided_value` fields in the error response schema use snake_case rather than the camelCase convention used throughout the rest of the API. This inconsistency will be corrected in a future version. Plan your error handling code accordingly.

See [Data models](/reference/reference-data-models#error-response-schema) for the full error response schema.

## Breaking changes in v2

### Title sorting removed from `GET /books`

The `GET /v2/books` endpoint no longer supports sorting results by title using `sort=title`. Use `sort=createdDate` instead. For migration steps, see [Migrate away from title sorting](/how-to-guides/how-to-guides-migrate-away-from-title-sorting).

### `Idempotency-Key` header

The POST /v2/books endpoint now requires an `Idempotency-Key` header to prevent duplicate book creation on retries.

### `409 Conflict` responses

Duplicate ISBNs are rejected with `409 Conflict`. v1 did not enforce ISBN uniqueness.

## Version 1.0 &mdash; Initial release

<Warning>v1 is supported until December 31, 2026. After this date, v1 endpoints will return `410 Gone`. Migrate to v2 before the deadline.</Warning>

The BookHub Publisher API v1 introduced programmatic management of book inventory on the BookHub marketplace, including endpoints for creating, retrieving, and managing books.

See [Migrate to BookHub Publisher API v2](/how-to-guides/how-to-guides-migrate-bookhub) for details on migrating from v1 to v2.
