## Version 2.0 — January 2, 2026

### New features

#### Hit count analytics

The `hitCount` field now shows how many times each book's detail page has been requested via the API, providing publishers with simple visibility into analytics. Use this metric to accurately gauge interest in specific titles. For example:

* Identify trending books before they become bestsellers.
* Measure marketing effectiveness by tracking post-campaign views.
* Make data-driven decisions about inventory and pricing.

**Endpoint:** `GET /v2/books/{bookId}`

**Response example:**

```json
{
  "status": "success",
  "data": {
    "bookId": "abc123",
    "title": "The Great Adventure",
    "author": "Jane Smith",
    "hitCount": 1247
  }
}
```

**Notes:**

- `hitCount` is read-only and automatically incremented each time the endpoint is accessed.
- The initial value for new books is `0`.
- Available in v2 only — not supported in v1.
- The count is per book and persists throughout the book's lifetime.

#### Delete a book

Publishers can now permanently remove inactive books from the BookHub marketplace, preventing them from appearing in all future queries.

**Endpoint:** `DELETE /books/{bookId}`

**Notes:**

- Only `INACTIVE` books can be deleted.
- This operation cannot be undone.
- Active books must be deactivated before deletion.
- Returns `204 No Content` on success.

### Breaking changes

#### Title sorting removed from `GET /books`

The `GET /v2/books` endpoint no longer accepts the `sort=title` parameter for sorting results by title.

To migrate, replace `sort=title` with `sort=createdDate`. For step-by-step migration instructions, see [Migrate away from title sorting](/how-to-guides/how-to-guides-migrate-away-from-title-sorting).
