Each BookHub API response includes the following rate limit headers, helping you monitor your current usage and limit status.

| Header Name | Description                                                                         |
|-------------|-------------------------------------------------------------------------------------| 
| `X-RateLimit-Limit` | The total number of requests allowed in the current time window (e.g., 150). |
| `X-RateLimit-Remaining` | The number of requests remaining in the current time window.            |
| `X-RateLimit-Reset` | The time when the current rate limit window resets, in UTC epoch seconds.   |

## Example response headers

```http
HTTP/1.1 200 OK
Content-Type: application/json
X-RateLimit-Limit: 150
X-RateLimit-Remaining: 120
X-RateLimit-Reset: 1700000000
```

In this example, you can send up to 150 requests per minute, leaving 120 remaining in the current window. If you exceed the limit, the API returns `429 Too Many Requests`. See [Monitor rate limits](/how-to-guides/how-to-guides-monitor-rate-limits) for retry guidance.
