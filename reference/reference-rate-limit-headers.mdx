title: 'Rate limit headers'
description: 'Understand how rate limit headers in API responses help you monitor and manage your BookHub API usage.'
---

The BookHub API includes rate limit headers in each response so you can monitor your current usage and limit status. The key headers to look for are:

| Header Name | Description |
|-------------|-------------| 
| `X-RateLimit-Limit` | The total number of requests allowed in the current time window (e.g., 150). |
| `X-RateLimit-Remaining` | The number of requests remaining in the current time window. |
| `X-RateLimit-Reset` | The time when the current rate limit window resets, in UTC epoch seconds. |

## Example response headers

```http
HTTP/1.1 200 OK
Content-Type: application/json
X-RateLimit-Limit: 150
X-RateLimit-Remaining: 120
X-RateLimit-Reset: 1700000000
```
In this example, you have a limit of 150 requests per hour, with 120 requests remaining. The `X-RateLimit-Reset` value indicates when the limit will reset, allowing you to plan your API usage accordingly.
