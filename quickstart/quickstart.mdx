## Introduction

This quickstart walks you through the core BookHub Publisher API workflow: creating a book, finalizing it for publication, and retrieving it to confirm it's live. By the end, you'll have a book listed as `ACTIVE` on the BookHub marketplace and a working foundation for building your integration.

## Prerequisites

<Prerequisites />

<Note> To create a BookHub Publisher account and obtain a JWT, email api-support@bookhub.com with your publisher information. </Note>

* A BookHub Publisher account. 
* A valid JWT for authentication. 
* An HTTPS client (curl, Postman, or programming language HTTP library).

## Step 1: Create a book (returns `PENDING` status)

Send a `POST` request to the `/v2/books` endpoint with the required fields to create a book. The status will be `PENDING` until finalized. For required and optional fields, see the [API endpoints](./reference/reference-api-endpoints#create-a-book).

To avoid creating duplicates, add an `Idempotency-Key` header with a unique value. Use `X-Request-ID` to track requests in logs and `X-Publisher-ID` to target a publisher account. See [Add the `Idempotency-Key` header](./how-to-guides/how-to-guides-add-the-idempotency-key-header) for details.

<IsbnNote />

```bash
curl -X POST https://api.bookhub.com/api/v2/books \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "title": "My New Book",
    "author": "John Doe",
    "description": "A new biography from a well-known author",
    "language": "en-US", 
    "publisher": "My Publishing House",
    "publishedDate": "2026-03-15",
    "isbn": "978-3-16-148410-0",
    "price": 24.99,
    "coverImages": ["https://example.com/cover.jpg"],
    "genres": ["Fiction", "Adventure"],
    "bookFormat": "Hardcover"
  }'
  ```

### Create book response (201 Created)

The response returns the new book’s details, including its `bookId`, which you can use in future requests. The initial status is `PENDING`. See the [API endpoints](./reference/reference-api-endpoints#error-codes) for error codes.

```json
{
  "status": "success",
  "message": "Book created successfully",
  "data": {
    "bookId": "bk_9x2m4k7p",
    "title": "My New Book",
    "author": "John Doe",
    "description": "A new biography from a well-known author",
    "language": "en-US",
    "publisher": "My Publishing House",
    "publishedDate": "2026-03-15",
    "isbn": "978-3-16-148410-0",
    "price": 24.99,
    "status": "PENDING",
    "coverImages": [
      "https://example.com/cover.jpg"
    ],
    "genres": [
      "Fiction",
      "Adventure"
    ],
    "bookFormat": "Hardcover",
    "createdDate": "2026-03-12T09:15:00Z",
    "updatedDate": "2026-03-12T09:15:00Z"
  }
}
```

## Step 2: Finalize the book (move from `PENDING` → `ACTIVE`)

Change the book’s status from `PENDING` to `ACTIVE` with this request. Use the `bookId` from the previous response (e.g., `bk_9x2m4k7p`). The `updatedDate` will reflect the change.

```bash
curl -X PATCH https://api.bookhub.com/api/v2/books/{bookId}/finalize \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"notes": "Ready for publication."}'
```

### Finalize book response (200 OK)

The response shows `ACTIVE` status. See the [API endpoints](./reference/reference-api-endpoints#error-codes) for error codes.

```json
{
  "status": "success",
  "message": "Book finalized successfully",
  "data": {
    "bookId": "bk_9x2m4k7p",
    "title": "My New Book",
    "status": "ACTIVE",
    "updatedDate": "2026-04-18T10:30:00Z"
  }
}
```

## Step 3: Get the book to see `hitCount` (starts at 0)

Send a `GET` request to `/v2/books/{bookId}` using the `{bookId}` value from the create step (e.g., `bk_9x2m4k7p`).

The response includes `hitCount`, starting at `0`. 
<Hitcount />
See the [API endpoints](./reference/reference-api-endpoints#error-codes) for error codes.

```bash
curl -X GET https://api.bookhub.com/api/v2/books/{bookId} \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"
```

## Step 4: List all books

Send a `GET` request to `/v2/books` to list all your books. Sort only by creation or update date in descending order.

<Note>Use `page` and `limit` query parameters for pagination. See [Handle pagination](./how-to-guides/how-to-guides-handle-pagination) for more information.</Note>

```bash
curl -X GET https://api.bookhub.com/api/v2/books?sort=createdDate&order=desc \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"
```

## Step 5: Test your integration

Verify your integration is working correctly by confirming the following:

* The create book request returns `201 Created` with a `bookId` in the response.
* The `status` field in the create response is `PENDING`.
* The `/v2/books/{bookId}/finalize` request returns `200 OK` with `status: ACTIVE`.
* The `/v2/books/{bookId}` request returns the book with `hitCount: 0`.
* The `/v2/books` request returns your new book in the results.

If any step fails, check the following:

* Your JWT is valid and included in the `Authorization` header.
* The `bookId` in Steps 2–3 matches the value returned in Step 1.
* The book status is `PENDING` before finalizing — calling finalize on an `ACTIVE` book returns a `400` error.

For a complete list of error codes, see [API endpoints](./reference/reference-api-endpoints#error-codes).
